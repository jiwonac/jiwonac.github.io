(Digital) Tools I Can't Live Without

I don't know where I first heard someone quip:

A good craftsman never blames his tools, for a good craftsman already has good tools.

That is, tools are rarely the defining differentiating factor between the novice and the master, though knowing and utilizing best tools does help. So here are the digital and productivity-related tools that I use at the moment!

My general philosophy for choosing my tools are summarized below.

  1. Segregate certain contexts into separate hardware.
  2. Store data in flat file formats.
  3. Use lightweight software for my old laptop.
  4. Prefer free and open source software without being a purist.
  5. Good default configurations are good.

With that said, let's start the list.

Hardware

  • Thinkpad t470s: My main work device.
    Although it's 5 years old at this point, it can do everything except graphically demanding tasks. Selling at around $200 USD refurbished, it fit well into a student budget after my old MacBook broke from water damage and I was looking for a laptop that will run Linux. I chose the t470s over the t470 since I heard there was an upgrade in build quality, which does matter for laptops.
  • Onyx Boox Max Lumi 2: The paper replacement e-ink tablet.
    As a student and researcher, I used to carry around heavy textbooks, binders full of papers and notepads everywhere. I replaced it all with the best 13-inch e-ink tablet available at the time. It matters that it is an e-ink device, since I mostly read and study with it; conventional tablets are far too distracting. Furthermore, it runs Android, which gives it much more flexibility over devices like Remarkable or a Kobo which runs highly locked down Linux.
  • Lami EMR Stylus: A nice writing experience on tablets that support Wacom EMR.
  • Steam Deck: The ultimate PC gaming console.
    I flat-out do not game on my (conventional) computers anymore. This thing can handle pretty much everything. I mean, c'mon, it runs Cyberpunk 2077! Although the console form factor makes it unsuitable for certain games that benefit from a keyboard and mouse setup, it has high extensibility which makes playing games like Frostpunk feasible as well. Unlike some other folks, I do not try to do anything other than gaming on this device. It's on darn good bang-for-buck gaming machine, and that's good enough for me.
  • SFF Desktop PC: Mini-ITX Gaming PC built with recycled parts.
    This device is a product of a time when I wanted a gaming PC that I could take anywhere, as I used to travel quite often. Now, the Steam Deck covers that use case. The Desktop PC mostly functions as a Blender machine at the moment. If I figure out a way to run blender with high performance on my laptop (cloud render farms? eGPU?) or on the Steam Deck then I might not even replace my PC when the current one finally dies. I built it with an RTX 580 GPU and an m.2 SSD salvaged from a dead eGPU and a dead laptop, among other parts.

I currently do not have a phone. I am still researching simple, small android devices.

Software

Foundational Software

  • Operating System: EndeavourOS (Arch Linux)
    Although I love Arch Linux for its unparalled software support through the arch user repository, installing Arch from scratch is bit of a hassle, even nowadays. EndeavourOS is my favorite way to have a smooth experience on Arch with minimal setup.
  • Desktop Environment: XFCE
    Since my Thinkpad is fairly old, more modern desktop environments such as GNOME, Budgie, and KDE run somewhat sluggishly. Among the lightweight DEs that I tested out, I liked XFCE the best for its easy GUI configuration, useful panel widgets, and a somewhat modern look and feel.
  • Web Browser: Firefox
    There still ain't better free and open-source and fully-featured web browser, and there likely never will be a real competitor for its niche.
  • Terminal Emulator: Alacritty
    It's fast. Whereas Gnome Terminal might have a bit of input lag on my Thinkpad, and xterm is a hassle to setup, Alacritty is snappy and pretty from the get-go.

Flatpak Packages

These are mostly apps that everyone else uses, so you have no choice but to use, or do not have suitable replacements on the Arch repo or AUR.

  • Discord: De-facto messaging app nowadays.
  • Slack: Necessary for work.
  • ONLYOFFICE Desktop: One of the few sensible Microsoft Office replacements on Linux.
  • Flatseal: Allows flatpak apps to gain more permissions.

Command-line Apps

  • Shell: fish
    Its default config behavior is much more user-friendly than bash, with convenient features such as suggestions and autocomplete.
  • Text editor: neovim + NvChad
    I could never get used to emacs, so vim it is. Neovim over vim any day simply due to its usage of lua rather than vimscript. NvChad is a very sensible default configuration for neovim which allows me to have a VSCode-like experience on neovim without too much hassle.
  • News reader: newsraft
    I found newsraft to be easy to configure, with a highly informative man page. It aims to be a simpler alternative to newsboat and I think it succeeds. I subscribe to video content creators, blogs, and podcasts.
  • Document viewer: zathura
    Is zathura a "true" command-line app? Probably not, though its minimal interface and vim-style navigation hotkeys certainly suit a terminal-centric workflow. My preferred way to view PDF or epub files.
  • Journaling & logging: jrnl
    A simple CLI application which automates plain-text journaling. I use it as my diary, "private Twitter" to write down showerthoughts, to record what I accomplished in a day, to keep track of workouts, etc.
  • Grammer checker: LanguageTool
    FOSS alternative to Grammarly that runs offline!

GUI Apps

  • Code editor: VSCodium
    A telemetry-free fork of VSCode. Useful for projects that are large enough that it makes sense to use a modern text editor or an IDE rather than neovim.
  • Media player: VLC
  • File synchronization: Syncthing-GTK
    An easy-to-use GUI frontend for Syncthing, a blazingly-fast file synchronization daemon. Although I know how to set up Syncthing using systemd, I can't be bothered. I rent a VPS which acts as a central cloud storage.
  • Torrenting: Transmission
    For downloading completely legal content such as Linux distros, of course. Although CLI alternatives exist, Transmission is intuitive and gets the job done.
  • TeX Match: $\LaTeX$ symbol search
  • Trimage: Lossless image compression
  • 3D Modelling: Blender

Web Apps

  • Overleaf: Cooperative $\LaTeX$ document editing
    Since everyone else uses it, and it allows for easier collaboration! Personally, I like to use their Git functionality to download the files offline rather than running their heavy webapp which makes my Thinkpad show its age.
  • Self-hosting: Yunohost
    This FOSS operating system makes it dead-simple to self-host web applications on a server. Could I do everything on my own? Probably. Though there's no way I'd keep up with the necessary security patches. I have good experience with the developers' support in their forums as well.
  • Reading log: Goodreads
    Although I tried several FOSS solutions in the past, it's really difficult to beat the extensive catalog that GoodReads has and its relatively straightforward UX at the same time.
  • Jupyter Lab: Data science & computer science stuff

But how do you...

  • Calendar: A cal.md file.
  • Todo list: A wm.md (working memory) file.
  • Note taking: Annotating a pdf template on my tablet (handwritten notes) or some markdown files (plain text notes). Pen & paper sometimes.
  • Reference manager: Plain bibtex files. It's not the best solution, but I dislike Zotero for other reasons.
  • Pretty much everything else: Simple nested file directories, flat files, and markdown files.

A useful tool for the flat-file lifestyle is the humble index.md file. Essentially, it bridges the gap between folders, which stores files, and markdown files, which stores notes. Storing some files in a directory, then adding an index.md file to take notes about what those files are for, allows us to simulate "notes with attachments" a la Evernote or OneNote without relying on external software.
